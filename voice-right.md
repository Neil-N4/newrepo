# Voice Right — user preferences

This file is read by `brain.py` and threaded into every style-transfer prompt. Edit
it freely — changes take effect on the next dictation. Keep it short and specific.

## My writing voice

- Direct, low on filler.
- Sentence fragments OK. Em dashes no, use commas or periods.
- Avoid corporate jargon ("leverage", "robust", "seamless", "comprehensive").
- Never use real people's names in examples or demos unless the name is already in the source material.

## App-specific rules

- **Gmail** — professional but human. Lead with the point. No "I hope this email finds you well."
- **iMessage** — lowercase start, casual, emojis OK if natural.
- **Discord** — short, fun, casual. Emojis encouraged.
- **LinkedIn** — measured, ambitious but not boastful. No "excited to announce" openers.
- **X / Twitter** — punchy, sometimes fragments. Max ~240 chars unless explicitly longer.
- **Terminal / shell** — commit messages: imperative mood ("add X", "fix Y"). Prose comments: short, one line.
- **Slack** — casual, functional, matches the channel's tone.

## Hard rules (never override)

- Never invent facts, numbers, or names. If the source is ambiguous, keep it ambiguous.
- Never add sign-offs ("Best,", "Cheers,") unless the source has one.
- Preserve meaning exactly. Only change form.
